# Part III: Structural Patterns

- Structural Patterns deal with how individual classes and objects are
  combined into complex objects

  - Class patterns look at the structures between classes
    - *Inheritance*
    - *Interfaces*
  - Object patterns look at how objects can be composed together into
    more complex objects
    - Object composition

- The pattern’s examined in this section are,

  1. The Adaptor Pattern
      - A technique for making one class interface match another
  2. The Composite Pattern
      - Objects are combined to create more complex objects
  3. The Proxy Pattern
      - A simplified interface takes the place of a more complex
        implementation
  4. The Flyweight Pattern
      - Each instance does not create it’s own state but refers to it
        externally
      - Object’s with the same state can then share that reference
      - Useful when there are many instances of an object, but most
        instances are the same
  5. The Facade Pattern
      - A single class is used to represent an entire complex subsystem
  6. The Bridge Pattern
      - The object’s interface is separated from the implementation
      - Allows the interface and implementation to be varied separately
  7. The Decorator Pattern
      - Allows dynamically adding behaviours to classes and methods
